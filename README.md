# Copilot Studio Artifacts

Welcome to the Copilot Studio Artifacts repository! This repository holds all artifacts for Copilot Studio generated apps and agents.

## 📁 Repository Structure

```
CopilotStudio/
├── apps/          # Copilot Studio generated applications
├── agents/        # Copilot Studio generated agents
├── shared/        # Shared resources and utilities
└── README.md      # This file
```

## 🎯 Purpose

This repository serves as a centralized location for:
- **Applications**: Complete applications generated by Copilot Studio
- **Agents**: AI agents and copilots created with Copilot Studio
- **Shared Resources**: Common utilities, templates, and configurations

## 🚀 Getting Started

### Adding a New App

1. Navigate to the `apps/` directory
2. Create a new subdirectory for your application
3. Add your application files and a README.md with documentation
4. See [apps/README.md](apps/README.md) for detailed guidelines

### Adding a New Agent

1. Navigate to the `agents/` directory
2. Create a new subdirectory for your agent
3. Add agent files including manifest, skills, and documentation
4. See [agents/README.md](agents/README.md) for detailed guidelines

### Using Shared Resources

The `shared/` directory contains reusable components and utilities. See [shared/README.md](shared/README.md) for more information.

## 📋 Guidelines

- **Organization**: Keep each app and agent in its own directory
- **Documentation**: Always include a README.md for each artifact
- **Naming**: Use clear, descriptive names for directories and files
- **Self-Contained**: Each artifact should be independent and self-contained
- **Version Control**: Commit meaningful changes with clear commit messages

## 🤝 Contributing

When contributing to this repository:

1. Follow the established directory structure
2. Include comprehensive documentation
3. Test your artifacts before committing
4. Keep sensitive data (keys, credentials) out of the repository
5. Use the .gitignore file to exclude build artifacts and dependencies

## 📖 Documentation

Each directory contains its own README.md with specific guidelines:

## 🔒 Security

- Never commit sensitive information (API keys, passwords, credentials)

- Review the .gitignore file to ensure sensitive files are excluded
